## How to Contribute

CampusOS welcomes contributions! Here's how you can help:

### Types of Contributions

1. **Code** — Backend modules, frontend components, bug fixes
2. **Documentation** — Improve docs, add examples, write guides
3. **Testing** — Write tests, report bugs, verify features
4. **Design** — UI/UX improvements, diagrams, visual design
5. **Research** — Case studies, competitive analysis, best practices

### Contribution Process

1. Read [COPILOT.md](../COPILOT.md) and .humanet/ for project context
2. Check [ROADMAP.md](../ROADMAP.md) and scope.md for current phase
3. Fork the repository
4. Create a feature branch (`git checkout -b feature/your-feature`)
5. Make your changes following module structure
6. Commit with clear messages
7. Push and create a Pull Request
8. Update Humanet documentation as needed

### Coding Guidelines

- Follow modular architecture (everything in `/apps/`)
- No direct module-to-module dependencies
- Update HUMANET and documentation
- Write tests for new features
- Keep controllers thin, business logic in services

## Karma System

Contributors earn karma points based on:
- Quality and impact of contributions
- Community recognition (upvotes)
- Consistency and engagement

Karma unlocks platform features and governance rights.
